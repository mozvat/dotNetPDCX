Welcome to the C# dsiPDCX SDK

This is a basic integration demonstrating how to initialize and perform a basic swiped credit sale utilizing the out-of-scope solution dsiPDCX from DataCap. http://www.datacapsystems.com/ This open-source project should be used as a supplement alongside the dsiPDCX integration guide from DataCap. If you need help or want to contribute, feel free to email me at mozvat@mercurypay.com

Basic overview

The DataCap dsiPDCX can be downloaded at: http://www.datacapepay.com/dsipdcx.htm

Below is the basic dataflow. Notice the communication of sensitive card data is contained within the dsiPDCX and the peripheral device. The dsiPDCX communicates with the light-weight 'In Store NetEpay' server. This server communicates with Mercury Payment Systems, (Or other payment processors). Your POS software interacts only with the dsiPDCX obtaining the results from the credit card processor.

Supported Devices

The dsiPDCX communicates with a variety of peripherals that accept a credit card swipe or manual entry. To get a list of the latest supported devices, request an integration guide from DataCap. Below shows a list of devices.

Basic Integration:

First, you have to reference the dsiPDCX.ocx within your project. Then perform a couple of initialization steps.

Notice you set the ServerIPConfig, this includes a back-up IP. TakeNote: dsiPDCX can be configured to communicate with the 'In-Store NetEPay' or directly to Mercury's NetEPay server. (Going directly to Mercury's NetEPay will require you handle your own logging and user-management of your POS)

  • Set the IPs. Call ServerIPConfig()
  • Set the timeouts. SetConnectionTimeOut(), SetResponseTimeOut()
  • Pass in the device initialize XML to the ProcessTransation() method.

Manual entry of credit card data

If the magstripe on the credit card is damaged and the device is unable to read a swipe. You can send a command to the IPAD to request manual entry of Credit Card information by passing in the keyword 'Prompt' in the AcctNo field. Depending on the device being used, the prompt for the credit card information will be obtained by the device.
